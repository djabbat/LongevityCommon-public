# DifferentiationAnnotation: Conceptual Framework

## §1 Purpose
Within Dr. Jaba Tkemaladze's CytogeneticTree project, the overarching goal is to reconstruct the complete genealogical tree of cellular differentiation from zygote to terminally differentiated cells by leveraging the asymmetric age of mother and daughter centrioles as a permanent lineage barcode. The **DifferentiationAnnotation** sub-subproject serves a critical interpretative function: it maps the raw lineage-directed acyclic graph (DAG), generated by tracking centriole age, onto biologically meaningful differentiation states. Its purpose is to transform a topological tree of "who begat whom" into an annotated tree of "what became what," where each node (cell) is labeled with its inferred cell state based on molecular and morphological signatures. This annotation is essential for understanding the timing, sequence, and potential bifurcation points of fate decisions embedded within the lineage tree.

## §2 Mechanism/Basis
The mechanism operates as a multi-modal data integration pipeline. The input is a lineage DAG where each node possesses a unique centriole-age-based identifier and temporal/spatial coordinates. Annotation proceeds via a rule-based and probabilistic fusion of orthogonal data streams:
1.  **scRNA-seq Marker Integration:** Single-cell RNA sequencing data from matched samples provides a high-dimensional transcriptomic profile. Cells within the lineage DAG are projected into this space using spatial/temporal registration or direct lineage barcode capture (e.g., intronic sequencing). Canonical marker genes for differentiation states are used to assign probabilistic labels (e.g., neural progenitor, cardiomyocyte precursor).
2.  **Morphology Feature Mapping:** High-resolution imaging data (e.g., light-sheet microscopy) provides cellular and nuclear morphology, cell volume, and neighbor topology. Pre-trained classifiers correlate specific morphological signatures with known cell states, offering an annotation layer independent of transcriptomics.
3.  **Immunofluorescence (IF) Anchor Points:** Sparse but definitive IF staining for key lineage-specific proteins (e.g., transcription factors, structural proteins) provides ground-truth anchor nodes. These anchors constrain and validate the annotations propagated from scRNA-seq and morphology through the lineage structure.

The core algorithmic challenge is the **constrained propagation** of state labels across the DAG. If a parent cell is annotated with high confidence, its daughter cells' possible states are constrained by known biological rules (e.g., a hepatoblast cannot directly produce a neuron). The lineage tree provides a powerful scaffold for smoothing and disambiguating state transitions that appear noisy in snapshot data.

## §3 State of the Art
Current methods for cell state annotation are largely divorced from lineage context. They operate on dissociated cells, losing the spatial and genealogical relationships that are fundamental to understanding differentiation dynamics.
*   **scRNA-seq Clustering & Annotation:** Standard pipelines (e.g., Seurat, Scanpy) cluster cells by transcriptomic similarity and annotate clusters using reference databases. These methods infer *pseudotime* but cannot reconstruct actual lineage trees (reference pending — general Seurat/Scanpy reviews; not verified here).
*   **Lineage Tracing with Molecular Barcodes:** Techniques like CRISPR-Cas9 barcoding (GESTALT, McKenna et al. 2016 Science, [PMID: 27229144]; scGESTALT, Raj et al. 2018 Nat Biotech, [PMID: 29608178]) can reconstruct lineages but typically lack simultaneous deep transcriptional phenotyping at single-cell resolution for all barcoded clones.
*   **Integrated Imaging & Sequencing:** Emerging spatial transcriptomics methods (MERFISH, seqFISH+) provide gene expression *in situ* but are limited in gene throughput and cannot yet trace deep historical lineages (reference pending — specific method paper not verified here).

**DifferentiationAnnotation** uniquely sits at the convergence of these approaches, using the intrinsic centriole-age lineage barcode to provide the temporal framework for integrating the deepest available molecular and morphological phenotyping.

## §4 Integration with Other CytogeneticTree Technologies
This subproject is a central hub for data synthesis:
*   **Input from `CentrioleLineageTracer`:** Consumes the raw lineage DAG (nodes, edges, timestamps).
*   **Input from `SingleCellBiopsySequencing`:** Receives transcriptomic profiles linked to specific lineage nodes via captured centriole identifiers or spatial mapping.
*   **Input from `LiveImagingMorphometry`:** Ingests time-lapsed morphological feature vectors for each tracked cell.
*   **Output to `FateDecisionMap`:** Provides the annotated lineage tree, which is the primary input for identifying bifurcation points and modeling fate decision logic.
*   **Output to `PhylogeneticReconstructionAlgos`:** Supplies state labels that can be used as character states for phylogenetic analysis of differentiation pathways.

## §5 Gaps & What to Build
Key gaps exist between the current state of art and the fully realized **DifferentiationAnnotation** system:
1.  **Spatio-Temporal Registration Engine:** No existing tool robustly aligns a live-imaging-derived lineage DAG with endpoint scRNA-seq data from dissociated cells. A probabilistic registration model must be built, potentially using the few IF anchor points as landmarks.
2.  **Lineage-Aware Annotation Propagator:** Current cell annotation tools are context-free. A novel algorithm is needed that propagates state labels across a DAG, respecting lineage relationships, temporal order, and permissible state transition rules (a "differentiation grammar").
3.  **Multi-Modal Confidence Integrator:** A framework to weigh and fuse conflicting evidence from transcriptomics, morphology, and IF, assigning a confidence score to each node's annotation is required.
**The build** must therefore focus on these three core software components: a Registration Engine, a Constrained Label Propagator, and a Confidence Integrator, forming the MVP of the annotation pipeline.
