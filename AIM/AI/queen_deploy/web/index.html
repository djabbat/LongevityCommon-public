How it works

  1. Worker (bee) runs AIM locally. Accumulates diagnostic runs, reflexion themes, skill usage, eval results.
  2. Periodically packages an anonymized signal — L_PRIVACY filter strips emails, phones, paths, names, PMIDs, DOIs before send. Posts to /v1/hive/contribute.
  3. Queen aggregates across workers. Detects cross-worker patterns: compliance drift, theme convergence, recurring failure clusters.
  4. Each candidate is eval-gated: Δscore ≥ 0.05, p ≤ 0.05, no regression on existing cases.
  5. Approved updates are signed and published to /v1/hive/updates. Workers pull, run three local gates: L_CONSENT (opt-out), signature integrity, L_VERIFIABILITY (re-eval).
  6. If all three pass — install. Otherwise log, continue offline.

Privacy contract

Workers send

  • aggregate counters (run count, retry rate)
  • numeric metrics (compliance %, avg crit)
  • anonymized theme labels (category words only)
  • hashed prompt fingerprint (sha256, never body)
  • worker_id (hash of hostname × per-install random salt)

Workers NEVER send

  • emails / phone numbers / contact info
  • file paths / project names
  • patient or clinical data
  • prompt/response content (only counts)
  • finding text (only file:line refs counted)

Install AIM locally

Each worker runs AIM on its own machine. We detected your OS; the command below clones the source from GitHub and bootstraps a local node (Ollama + venv + ~/.aim_env). 

curl -L -o aim.tar.gz https://github.com/djabbat/AIM-public/releases/latest/download/aim-0.1.0-linux.tar.gz
tar -xzf aim.tar.gz
cd aim-0.1.0-linux
bash scripts/install_node.sh
curl -L -o aim.tar.gz https://github.com/djabbat/AIM-public/releases/latest/download/aim-0.1.0-macos.tar.gz
tar -xzf aim.tar.gz
cd aim-0.1.0-macos
bash scripts/install_node.sh
Invoke-WebRequest -Uri https://github.com/djabbat/AIM-public/releases/latest/download/aim-0.1.0-windows.zip -OutFile aim.zip
Expand-Archive aim.zip
cd aim-0.1.0-windows
powershell -ExecutionPolicy Bypass -File scripts\install_node.ps1

Connect to the hive

# 1. Confirm DEEPSEEK_API_KEY etc. set in ~/.aim_env
aim setup-key

# 2. Add the queen URL
echo 'AIM_HIVE_QUEEN_URL=https://hive.longevity.ge' >> ~/.aim_env

# 3. Preview what would send (dry run, NOTHING transmitted)
aim diag --hive-preview

# 4. Status
aim diag --hive-status
