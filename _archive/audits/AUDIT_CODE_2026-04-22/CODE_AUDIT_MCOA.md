# Code Audit — MCOA

## ✅ Соответствия
1.  **Параметры в коде:** В ядре (`mcoa_core`) реализованы функции для получения параметров (`default_reference_scales`, `default_drift_rates`, `default_weights`), которые, судя по вызовам и именам, должны соответствовать таблицам в `PARAMETERS.md`.
2.  **Формулы в `CONCEPT.md`:** Базовая формула для независимого дрейфа повреждения `D_i(n, t)` корректно реализована в функции `independent_drift` в `mcoa_core/src/lib.rs`. Она соблюдает аксиому M2 о размерной согласованности, нормируя количество делений и время.
3.  **Отсутствие устаревших утверждений:** В проверенном коде отсутствуют прямые упоминания отозванных концепций:
    *   Нет формулы `Health Score = 0.40*organism+...`.
    *   Нет ссылок на "MCOA Test 2" как источник параметров.
    *   Нет утверждений о `χ_Ze R²=0.84` или "validated biomarker".
    *   Нет упоминаний Ze/BioSense/Aqtivirebuli как рабочих пакетов EIC.
4.  **Логика сравнения:** Реализованы скрипты `compare_mcoa_cdata.py` и `compare_all.py`, которые автоматизируют обязательное правило сравнения выходных данных MCOA и CDATA, что соответствует документации.
5.  **Scaffold-тесты:** Модули `test1_dominance` и `test4_aubrey` созданы как заглушки для канонических тестов фальсифицируемости, упомянутых в `CONCEPT.md`.

## ❌ Проблемы (Критические несоответствия канону)
1.  **Нарушение канона: γ_i ≠ 0 по умолчанию.**
    *   **Файл:** `crates/mcoa_simulation/src/lib.rs`
    *   **Строка ~66:** `let gamma_i = 0.01; // per-step coupling strength scalar`
    *   **Несоответствие:** Согласно `CORRECTIONS_2026-04-22` (п. 1.3), по умолчанию должно быть `γ_i = 0` (гипотеза независимости). Ненулевые значения могут появиться только после пост-фактум статистического анализа. Установка `0.01` по умолчанию нарушает этот канон и вносит априорную связь между счетчиками без обоснования.
    *   **Контекст:** Эта константа используется для расчета вклада связи (`coupling`) на каждом шаге симуляции.

2.  **Неполная реализация канонических параметров.**
    *   **Файл:** `crates/mcoa_simulation/src/lib.rs`
    *   **Проблема:** В функции `step` используется объект `Gamma` (матрица связи), но не проверено, что реализация `Gamma::default()` возвращает нулевую матрицу (как должно быть по новому канону). Также отсутствует явная загрузка матрицы `Γ` из `PARAMETERS.md` §4.

## ⚠️ Улучшения (Некритичные замечания)
1.  **Ссылки на канонические документы:** В комментариях к коду есть ссылки на `CONCEPT.md`, но нет явных отсылок к `CORRECTIONS_2026-04-22.md` — единому источнику истины. Это может привести к тому, что разработчики будут руководствоваться устаревшими утверждениями из других документов.
2.  **Проверка весов тканей:** Функция `TissueWeights::is_normalised` проверяет сумму весов, но не вызывается автоматически при использовании `default_weights`. Хорошей практикой была бы валидация на старте симуляции.
3.  **Статус кода:** Значительная часть кода, особенно в `mcoa_tests`, отмечена как `TODO` или является заглушкой. Это соответствует текущей стадии проекта, но требует пометки в аудите.

**Итог:** Кодовая база MCOA в основном структурно соответствует core-документам, однако содержит **критическое отклонение** от установленного канона (`CORRECTIONS_2026-04-22`) в отношении значения параметра связи γ_i. Остальные несоответствия носят характер недостаточной валидации или являются следствием незавершенной (scaffold) реализации. Рекомендованные правки минимальны и напрямую исправляют нарушение канона.
