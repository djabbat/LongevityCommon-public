# Code Audit — Proteostasis

## ✅ Соответствия

1. **Формула повреждения (CONCEPT.md ↔ код):** Основное уравнение `D₅(n, t) = D₅,₀ + α₅·(n / n₅*) + β₅·(t / τ₅) + γ₅·I(others)` корректно реализовано в функции `compute_damage` (`src/lib.rs:52-56`). Структура `CounterParams` содержит все соответствующие поля.

2. **Канонические определения (CORRECTIONS_2026-04-22):**
    * В коде **отсутствуют** упоминания "Health Score", "MCOA Test 2", `χ_Ze` как биомаркера (R²=0.84) — требования §1.1, §1.2, §1.3 соблюдены.
    * Параметр `gamma` по умолчанию установлен в `0.0` (`src/lib.rs:34`), что соответствует требованию `γ_i = 0 by default` (§1.3).
    * Код не касается определения `v` (Ze-скорости), конфликта нет.

3. **Логика тканевой специфичности (CONCEPT.md ↔ код):** Модуль `tissue` (`src/tissue.rs`) реализует концепцию различных параметров для пролиферирующих и постмитотических тканей (например, снижение `alpha` для нейронов), что соответствует биологическому обоснованию из CONCEPT.md, раздел 2.2.

4. **Соответствие параметров (частичное):** Параметры `D₅,₀` (`d0`), `α₅` (`alpha`), `β₅` (`beta`), `n₅*` (`n_star`), `τ₅` (`tau_days`), `γ₅` (`gamma`), `d_critical` из PARAMETERS.md присутствуют в структуре `CounterParams`. Базовые веса тканей `w₅` реализованы методом `Tissue::weight()`.

## ❌ Проблемы

1. **Критическое несоответствие числовых параметров (PARAMETERS.md ↔ код):**
    * `src/lib.rs:34` (дефолтные параметры):
        * `n_star = 80.0`, в то время как в PARAMETERS.md указано `n₅* = 50` (для фибробластов).
        * `tau_days = 29200.0` (80 лет), в то время как в PARAMETERS.md указано `τ₅ = 10` лет.
        * `alpha = 0.2795`, `beta = 0.2795`, в то время как в PARAMETERS.md: `α₅ = 0.05`, `β₅ = 0.1`.
    * **Вывод:** Дефолтные значения в коде **не соответствуют** таблице PARAMETERS.md. Код использует значения, полученные скриптом калибровки (`scripts/calibrate.py`), который опирается на другие литературные анкеры. Это создает конфликт между документацией и исполняемой логикой.

2. **Неполная реализация системы связи (γ):**
    * В PARAMETERS.md описаны пять отдельных коэффициентов связи `γ₅₁`...`γ₅₅` для влияния других счетчиков и автокаталитического эффекта.
    * В коде (`src/lib.rs:27`) используется единый параметр `gamma`, что является грубым упрощением и не позволяет моделировать специфические взаимодействия, описанные в документации (KNOWLEDGE.md, раздел "Механистические связи").

3. **Расхождение в весах тканей (PARAMETERS.md ↔ код):**
    * В PARAMETERS.md приведены веса для neuron (0.4), muscle (0.2), liver (0.05).
    * В коде (`src/tissue.rs:33`) вес для `Tissue::Neuron` установлен в 0.2, для `Tissue::Cardiomyocyte` (мышца) — 0.15, для `Tissue::Hepatocyte` (печень) — 0.2.
    * **Вывод:** Конкретные числовые веса в коде **не совпадают** с гипотетическими значениями из PARAMETERS.md, что может привести к путанице при калибровке.

4. **Отсутствие параметра `d_critical` в PARAMETERS.md:** В таблице параметров отсутствует пороговое значение `d_critical`, хотя оно используется в коде (`src/lib.rs:28`) и логике (`is_above_critical`, `divisions_to_critical`).

## ⚠️ Улучшения

1.  **Структура связи:** Текущая реализация `coupling` как одного числа — слишком абстрактна. Для исследовательской ценности лучше реализовать вектор `gamma` или отдельные поля `gamma_1..gamma_5`, даже если по умолчанию они равны 0.
2.  **Документация в коде:** В `src/lib.rs` можно добавить комментарии, прямо ссылающиеся на уравнения из CONCEPT.md и правила CORRECTIONS_2026-04-22 (особенно про `gamma = 0`).
3.  **Валидация параметров:** Метод `CounterParams::validate` можно расширить, чтобы проверять, что `alpha` и `beta` не равны нулю одновременно, или что `d_critical > d0`.
4.  **Скрипт калибровки:** В `scripts/calibrate.py` константы `N_STAR_DIV = 80.0` и `TAU_YEARS = 80.0` жестко закодированы, но их следовало бы брать из PARAMETERS.md или делать входными аргументами.
